03 Architecture

시스템 전체 구조

플랫폼은 학원·서점만 대상으로 하며, 자격검증·전자서명·결제·택배·정산·LMS를 위해 외부와 연동합니다. 지사·총판은 제외(점선)되어 기존 ERP 직발주로 유지됩니다.

그림 2. 전체 구조 — 대상: 학원·서점 / 제외: 지사·총판(점선) / NE Tutor 연동
04 Common Backbone

가입 · 자격검증 · 전자계약 (공통)

학원·서점 공통 핵심 흐름. 국세청 API 자격검증과 전자서명 계약(효력 발생)이 핵심이며, 약관법 리스크를 반영한 동의 UI를 적용합니다.

그림 3. 공통 플로우 — 가입→자격검증→전자계약(학원=제휴/서점=공급)
국세청 API 자격검증

진위확인+상태조회로 자동 판정. 서점은 재판매 업종 확인, 지사·총판 거래 중복 대조(공정거래법 차단). 분기 1회 재검증.

전자서명 계약 (효력 발생)

본질조건 특정 + 동의 별첨 편입 + 권한자 전자서명 → 서명 시점에 효력 발생. 네이버폼만으로는 효력 불충분(6/26).

동의 방식
개별 체크박스 Opt-in, "제출 시 동의 간주" 삭제, 개인정보 동의 별도(개인정보보호법), "정식계약 체결 시 편입·효력" 안내, 신청취소 창구
권한 확인
대표 본인/위임 직원(재직증명서)+신분확인 고지
통합회원(학원 학생)
학생·학부모 통합회원 가입(개인정보 본인 입력), 미성년 결제는 법정대리인 동의(법무 확인)
05 Academy · Link & LMS

학원 — 링크공유 · LMS (NE Tutor)

선생님이 반을 만들고 학생을 임의 등록한 뒤 링크를 전달 → 학생이 링크로 들어와 통합회원 가입 후 구매 → 구매와 동시에 NE Tutor 콘텐츠 이용권 부여. 이를 위해 LMS(반–학생–교재–콘텐츠) 구조를 사전 설계합니다.

05-1 링크공유 → 통합회원 구매 → NE Tutor 이용
그림 4. 학원 링크공유 — 선생님(반·학생·링크)→학생(통합회원·구매)→시스템(LMS·NE Tutor 이용권). 학원은 수수료 정산.
05-2 LMS 데이터 구조 (사전 설계 · IT 기획자용)
그림 5. LMS 엔터티 — 학원→반→학생(통합회원)→주문→콘텐츠 이용권→NE Tutor (구매 시 이용권 자동 발급)
기능상세우선
반·학생 관리(LMS)반 생성, 학생 임의 등록(개인정보 미입력), 반-학생 매핑필수
구매 링크반·교재 지정→링크 자동 생성·공유, 진입 시 반·교재 고정필수
통합회원 가입본인 정보 직접 입력, 미성년 법정대리인 동의 처리필수
결제·배송정가 15% 내 결제, 택배, 청약철회필수
콘텐츠 이용권·NE Tutor구매 시 이용권 자동 발급·학생 계정 연결, 연동 포인트 사전 설계사전설계
수수료 정산학원별 판매액 집계→제휴 수수료 월 정산중요
법적 포인트(학원) 학원은 링크 공유·수수료만 → 학원법 회피. 학생거래는 NE능률이 판매자인 소매(정가제 15% 준수).
법무 확인 필요(신규) ① NE Tutor 콘텐츠 무료/번들 제공 시 경제상의 이익 한도 합산 여부 ② 미성년 결제·법정대리인 동의 ③ 학원 제휴 수수료의 학원법 적합성 ④ 통합회원 약관. (메뉴 07 반영)
06 Bookstore

서점 — 도매 직거래 · 지사 해지 전환

서점은 사업자로서 도매가(공급율 67%)로 매입해 재판매합니다. 지사 해지로 거래가 끊길 서점을 본사 직거래로 흡수하며, 거래 공백 방지를 위해 해지 전 전환(선전환)을 원칙으로 합니다.
